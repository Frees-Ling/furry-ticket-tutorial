# Windows Docker 部署指南

## 1. Docker Desktop for Windows

### 1.1 系统要求

- Windows 10 64-bit: Pro, Enterprise, or Education (Build 19044+)
- Windows 11 64-bit: Home or Pro
- 启用 WSL 2（推荐）或 Hyper-V
- 4GB+ RAM（推荐 8GB+）

### 1.2 安装 WSL 2 后端（推荐）

```powershell
# 1. 启用 WSL
wsl --install

# 2. 设置 WSL 2 为默认版本
wsl --set-default-version 2

# 3. 安装 Ubuntu 发行版
wsl --install -d Ubuntu

# 4. 验证
wsl -l -v
# 输出示例：
#   NAME      STATE           VERSION
#   Ubuntu    Running         2
```text

### 1.3 安装 Docker Desktop

```powershell
# 下载 Docker Desktop for Windows
# https://docs.docker.com/desktop/install/windows-install/

# 安装时选择 "Use WSL 2 instead of Hyper-V"
# 安装完成后重启

# 验证安装
docker --version
docker compose version

# 设置 Docker Desktop 开机自启
# Settings → General → "Start Docker Desktop when you sign in"
```

## 2. 配置 Docker Desktop

### 2.1 资源分配

通过 Docker Desktop GUI 调整：

```text
Settings → Resources → Advanced
  CPUs: 4 (根据硬件调整)
  Memory: 4GB (推荐 4-8GB)
  Swap: 2GB
  Disk image size: 64GB
```

### 2.2 WSL 2 集成

```text
Settings → Resources → WSL Integration
  ✅ Enable integration with my default WSL distro
  ✅ Ubuntu (或你安装的发行版)
```

### 2.3 配置镜像加速（国内用户）

```powershell
# Docker Desktop → Settings → Docker Engine
# 添加 registry-mirrors 配置
```text

```json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ]
}
```

## 3. 启动项目服务

### 3.1 启动 MySQL + Redis

```powershell
# 在项目根目录执行
cd C:\ticket-booking

# 启动开发环境数据库
docker compose up -d

# 查看容器状态
docker ps

# 预期输出两个容器：
# - ticket-mysql:  运行在 3306 端口
# - ticket-redis:  运行在 6379 端口

# 验证 MySQL 连接
docker exec -it ticket-mysql mysql -uroot -proot123 ticket_dev -e "SELECT 1"

# 验证 Redis 连接
docker exec -it ticket-redis redis-cli ping
# 应返回 PONG
```text

### 3.2 常见端口冲突解决

如果本地已安装 MySQL/Redis 导致端口冲突：

```powershell
# 方案1：停止本地服务
Stop-Service MySQL80       # 停止本地 MySQL
Stop-Service Memurai       # 停止本地 Memurai/Redis

# 方案2：修改 docker-compose 端口映射
# docker-compose.yml 中修改：
#   ports:
#     - "3307:3306"   # 主机 3307 → 容器 3306
#     - "6379:6379"   # 保持默认

# 方案3：使用不同的 compose 文件
docker compose -f docker-compose.yml up -d
```

### 3.3 启动生产环境

```powershell
# 生产环境启动所有服务（包括 API + Worker）
docker compose -f docker-compose.prod.yml up -d

# 查看所有容器状态
docker compose -f docker-compose.prod.yml ps

# 查看实时日志
docker compose -f docker-compose.prod.yml logs -f

# 查看特定服务日志
docker compose -f docker-compose.prod.yml logs -f api
docker compose -f docker-compose.prod.yml logs -f worker
```text

## 4. 跨文件路径映射

### 4.1 WSL 文件系统访问

```powershell
# Windows 路径在 WSL 中的映射
# C:\ticket-booking → /mnt/c/ticket-booking

# 在 WSL 终端中访问项目
wsl
cd /mnt/c/ticket-booking
ls -la

# 在 WSL 中执行 Docker 命令（比 PowerShell 更快）
docker compose up -d
```

### 4.2 卷挂载注意事项

```yaml
# docker-compose.yml 中的卷挂载
volumes:
  - ./alembic/versions:/app/alembic/versions  # 确保使用相对路径
```text

Windows 路径注意事项：

- 始终使用 `/` 而不是 `\`（Docker 内部使用 Linux 路径）
- `C:\path` 应写为 `/c/path` 或 `./relative/path`
- MySQL 数据卷建议放在 WSL 2 文件系统内（性能更好）

## 5. 性能优化

### 5.1 将项目放在 WSL 2 文件系统内

```powershell
# 在 WSL 2 中克隆项目（比在 Windows 文件系统快 3-5 倍）
wsl
cd ~
git clone https://github.com/your-org/ticket-booking.git
cd ticket-booking
docker compose up -d
```

### 5.2 避免文件系统性能问题

```text
❌ 不要这样做：在 Windows 文件系统（/mnt/c/）中运行 Docker
   → 文件监听缓慢，卷挂载性能差
   
✅ 推荐做法：在 WSL 2 原生 ext4 文件系统中运行
   → 原生 Linux 性能
```

## 6. 常见问题

### 6.1 Docker Desktop 无法启动

```powershell
# 清除 WSL 2 发行版缓存
wsl --shutdown

# 重启 Docker Desktop
# 或重置网络：
netsh winsock reset

# 如果问题持续，尝试：
# Settings → Troubleshoot → Reset to factory defaults
```text

### 6.2 容器无法访问

```powershell
# 检查端口映射
docker ps

# 检查容器日志
docker logs ticket-mysql --tail 50
docker logs ticket-redis --tail 50

# 重启容器
docker restart ticket-mysql
docker restart ticket-redis
```

### 6.3 磁盘空间不足

```powershell
# 清理未使用的容器、镜像、卷
docker system prune -a --volumes

# 在 Docker Desktop 中压缩磁盘：
# Settings → Resources → Advanced → "Disk image size" 或
# Troubleshoot → "Clean / Purge data"
```text

### 6.4 WSL 2 磁盘占用过大

```powershell
# WSL 2 虚拟磁盘自动扩展但不自动收缩
# 手动压缩（以 Ubuntu 为例）：
wsl --shutdown
diskpart
# 在 diskpart 中：
select vdisk file="C:\Users\lenovo\AppData\Local\Docker\wsl\data\ext4.vhdx"
attach vdisk readonly
compact vdisk
detach vdisk
exit
```
