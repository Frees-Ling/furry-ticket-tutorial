# 测试指南

## 测试架构

本项目采用**双层测试架构**：Mock 测试（默认）和 集成测试（`--integration`）。

```text
┌──────────────────────────────────────────────┐
│              测试套件                           │
│                                                  │
│  ┌─────────────────┐  ┌────────────────────┐   │
│  │ Mock 测试 (默认)  │  │ 集成测试 (--integration)│   │
│  │                  │  │                    │   │
│  │ 无外部依赖        │  │ 需要 MySQL + Redis │   │
│  │ 快速运行 (~5s)    │  │ 较慢但更真实       │   │
│  │ 覆盖边界/异常场景  │  │ 覆盖业务流程/并发   │   │
│  └─────────────────┘  └────────────────────┘   │
└──────────────────────────────────────────────┘
```

## 运行测试

### 快速运行（Mock 测试，默认）

```bash
# 全部 Mock 测试
pytest

# 显示详细输出
pytest -v

# 运行单个测试文件
pytest tests/test_auth.py -v

# 运行单个测试函数
pytest tests/test_auth.py::test_login_success -v

# 带覆盖率报告
pytest --cov=app --cov-report=term-missing --cov-fail-under=80 -v
```text

### 集成测试（需要外部服务）

```bash
# 1. 启动 MySQL + Redis
docker compose up -d

# 2. 运行所有测试（Mock + 集成）
pytest --integration -v

# 3. 仅运行集成测试
pytest --integration tests/test_integration.py -v

# 4. 运行并发防超卖测试
pytest --integration tests/test_concurrent.py -v -s
```

**注意：** 集成测试使用事务回滚隔离，每个测试结束后自动回滚，不会污染数据库。

### CI/CD 中的测试

在 GitHub Actions 中，使用 `services` 启动 MySQL 和 Redis 容器后运行集成测试：

```yaml
- name: Run tests
  run: |
    pytest --integration --cov=app --cov-report=term-missing -v
```text

详见 `.github/workflows/test.yml`。

## 测试文件结构

| 文件 | 类型 | 测试内容 |
| ------ | ------ | --------- |
| `test_auth.py` | Mock | 注册/登录/登出/Token 刷新 |
| `test_events.py` | Mock | 活动 CRUD 路由注册、参数验证 |
| `test_orders.py` | Mock | 订单创建/取消/列表/自动取消 |
| `test_admin.py` | Mock | 管理员封号/解封/用户管理 |
| `test_payments.py` | Mock | 支付通知处理 |
| `test_anti_overselling.py` | Mock | 防超卖 Lua 脚本逻辑 |
| `test_integration.py` | 集成 | 端点到端点的业务流程测试 |
| `test_concurrent.py` | 集成 | 并发抢票防超卖验证 |

## 测试质量要求

- 新功能必须包含测试
- Mock 测试覆盖边界条件和异常路径
- 关键业务逻辑（防超卖、支付幂等性）必须有集成测试
- 测试覆盖率目标：`>= 80%`（通过 `--cov-fail-under=80` 强制检查）

## Fixture 说明

### Mock 测试 Fixtures（`tests/conftest.py`）

| Fixture | 用途 |
| --------- | ------ |
| `mock_db_session` | Mock 的 AsyncSession，所有方法返回默认值 |
| `mock_redis` | Mock 的 Redis 客户端，支持常用命令 |
| `client` | httpx.AsyncClient，使用 ASGITransport 直连 |
| `auth_token` | 模拟的 JWT Token 字符串 |
| `mock_user` | 模拟的普通用户对象 |
| `mock_admin_user` | 模拟的管理员用户对象 |

### 集成测试 Fixtures（`tests/conftest_db.py`）

| Fixture | 用途 |
| --------- | ------ |
| `db_engine` | 真实数据库引擎（session 级别复用） |
| `db_session` | 真实数据库会话（事务回滚隔离） |
| `redis_client` | 真实 Redis 连接 |
| `integration_client` | 使用真实 DB/Redis 的 HTTP 客户端 |
| `test_user` | 预创建的测试用户 |
| `test_admin` | 预创建的测试管理员 |
| `test_event` | 预创建的测试活动 |
| `auth_token` | 预获取的 JWT Token |
