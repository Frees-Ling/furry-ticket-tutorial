## 9.5.1 Postman 集合概览

本项目提供了一个完整的 Postman 集合文件 `postman/ticket-system.postman_collection.json`，包含了所有 API 端点的测试请求。

### 集合结构

```text
票务系统 API
├── 1. 认证（Auth）
│   ├── 注册 (POST /api/v1/auth/register)
│   ├── 登录 (POST /api/v1/auth/login)
│   ├── 刷新 Token (POST /api/v1/auth/refresh)
│   └── 登出 (POST /api/v1/auth/logout)
│
├── 2. 活动（Events）
│   ├── 创建活动 (POST /api/v1/events)
│   ├── 活动列表 (GET /api/v1/events)
│   ├── 活动详情 (GET /api/v1/events/{id})
│   ├── 更新活动 (PUT /api/v1/events/{id})
│   └── 删除活动 (DELETE /api/v1/events/{id})
│
├── 3. 订单（Orders）
│   ├── 创建订单 (POST /api/v1/orders)
│   ├── 我的订单 (GET /api/v1/orders)
│   ├── 取消订单 (POST /api/v1/orders/{id}/cancel)
│   └── 订单详情 (GET /api/v1/orders/{id})
│
├── 4. 支付（Payments）
│   ├── 发起支付 (POST /api/v1/payments/pay)
│   ├── 支付结果查询 (GET /api/v1/payments/{order_no})
│   └── 支付宝异步通知 (POST /api/v1/payments/alipay/notify)
│
├── 5. 管理（Admin）
│   ├── 封禁用户 (POST /api/v1/admin/users/{id}/ban)
│   ├── 解封用户 (POST /api/v1/admin/users/{id}/unban)
│   ├── 用户列表 (GET /api/v1/admin/users)
│   └── 删除活动 (DELETE /api/v1/admin/events/{id})
│
└── 6. 健康检查
    ├── 健康检查 (GET /health)
    └── 就绪检查 (GET /health/ready)
```

## 9.5.2 导入与配置

### 导入集合

1. 打开 Postman
2. 点击 **File → Import**（或按 `Ctrl+O`）
3. 选择 `postman/ticket-system.postman_collection.json`
4. 点击 **Import** 确认

### 配置环境变量

1. 点击 Postman 右上角的齿轮图标 → **Add**
2. 创建新环境 `ticket-system-local`
3. 添加以下变量：

| 变量名 | 初始值 | 说明 |
| -------- | -------- | ------ |
| `base_url` | `http://localhost:8000/api/v1` | API 基础地址 |
| `token` | （空） | 登录后自动填充 |
| `event_id` | （空） | 创建活动后自动填充 |
| `order_id` | （空） | 创建订单后自动填充 |
| `order_no` | （空） | 支付接口使用 |

> **注意**：`token` 变量通过**登录接口的 Pre-request Script** 自动设置，无需手动填写。

## 9.5.3 Pre-request Script 详解

Pre-request Script 是 Postman 的强大功能，可以在请求发送前执行 JavaScript 代码。

### 登录接口的自动 Token 设置

```javascript
// postman 集合中登录请求的 Pre-request Script
const email = pm.environment.get("login_email") || "test@example.com";
const password = pm.environment.get("login_password") || "test123456";

pm.environment.set("login_email", email);
pm.environment.set("login_password", password);
```text

### 登录响应的自动 Token 提取（Tests 脚本）

```javascript
// 登录请求的 Tests 脚本：自动提取 Token
const response = pm.response.json();

if (pm.response.code === 200) {
    // 保存 access_token 到环境变量（后续所有请求自动携带）
    pm.environment.set("token", response.access_token);

    // 保存 refresh_token
    pm.environment.set("refresh_token", response.refresh_token);

    // 保存用户信息
    pm.environment.set("user_id", response.user.id.toString());

    console.log("Token 已自动保存");
}
```

### 通用 Auth 脚本

集合级别的 **Pre-request Script** 会自动将 Token 注入到所有需要认证的请求头中：

```javascript
// 集合级别 Pre-request Script
const token = pm.environment.get("token");
if (token) {
    pm.request.headers.add({
        key: "Authorization",
        value: `Bearer ${token}`
    });
}
```text

## 9.5.4 测试流程

### 第一步：注册用户

```

POST {{base_url}}/auth/register
Content-Type: application/json

{
    "username": "test_user",
    "email": "<test@example.com>",
    "password": "test123456"
}

```text

**预期响应**：`201 Created`

```json
{
    "id": 1,
    "username": "test_user",
    "email": "test@example.com",
    "role": "user"
}
```

### 第二步：登录获取 Token

```text
POST {{base_url}}/auth/login
Content-Type: application/json

{
    "email": "test@example.com",
    "password": "test123456"
}
```

**预期响应**：`200 OK`

```json
{
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "token_type": "bearer",
    "user": { "id": 1, "username": "test_user", "role": "user" }
}
```text

**关键点**：Tests 脚本自动提取 `access_token` 保存到 `{{token}}` 环境变量，后续请求无需手动配置。

### 第三步：创建活动（需要 admin 角色）

> **注意**：普通用户无法创建活动，需要先通过数据库将用户角色修改为 `admin`，或使用已创建的管理员账号。

```

POST {{base_url}}/events
Authorization: Bearer {{token}}
Content-Type: application/json

{
    "title": "周杰伦2026演唱会",
    "description": "一场精彩绝伦的演唱会",
    "venue": "国家体育场（鸟巢）",
    "start_date": "2026-12-01T19:00:00",
    "end_date": "2026-12-01T23:00:00",
    "category": "concert",
    "price": 680.00,
    "total_stock": 1000
}

```text

**Tests 脚本**：自动提取 `event.id` 保存到 `{{event_id}}`

### 第四步：查看活动列表

```

GET {{base_url}}/events?page=1&size=10

```text

**预期响应**：

```json
{
    "items": [{"id": 1, "title": "周杰伦2026演唱会", ...}],
    "total": 1,
    "page": 1,
    "size": 10,
    "total_pages": 1
}
```

### 第五步：创建订单

```text
POST {{base_url}}/orders
Authorization: Bearer {{token}}
Content-Type: application/json

{
    "event_id": 1,
    "quantity": 2
}
```

**预期响应**：`201 Created`

```json
{
    "id": 1,
    "order_no": "20261201123456",
    "event_id": 1,
    "quantity": 2,
    "total_amount": "1360.00",
    "status": "pending"
}
```text

**Tests 脚本**：自动提取 `order.id` 和 `order.order_no` 保存到环境变量。

### 第六步：发起支付

```

POST {{base_url}}/payments/pay
Authorization: Bearer {{token}}
Content-Type: application/json

{
    "order_no": "20261201123456",
    "pay_method": "alipay"
}

```text

**预期响应**：返回支付宝支付链接（form 表单 HTML）。

### 第七步：查询订单列表

```

GET {{base_url}}/orders
Authorization: Bearer {{token}}

```text

**预期响应**：包含该用户所有订单的分页列表。

## 9.5.5 集合变量与数据流

Postman 集合通过 **Tests 脚本** 在请求间传递数据：

```

注册 → 登录 → 获取 token（自动保存）
  │
  ├──→ 创建活动 → 获取 event_id（自动保存）
  │      └──→ 查询活动列表
  │
  └──→ 创建订单 → 获取 order_id, order_no（自动保存）
         ├──→ 发起支付
         ├──→ 查询订单列表
         └──→ 取消订单

```text

这种链式数据流让你可以一键运行整个集合，逐个执行请求即可完成完整业务流程。

## 9.5.6 Newman 命令行自动化

Newman 是 Postman 的命令行工具，可将集合集成到 CI/CD 流程中。

### 安装 Newman

```bash
npm install -g newman
```

### 运行完整集合

```bash
newman run postman/ticket-system.postman_collection.json \
  --environment postman/ticket-system.postman_environment.json \
  --reporters cli,json \
  --reporter-json-export newman-report.json
```text

### 带数据的批量测试

```bash
newman run postman/ticket-system.postman_collection.json \
  --iteration-count 10 \
  --delay-request 100 \
  --timeout-request 5000
```

参数说明：

- `--iteration-count`：迭代次数（模拟多次下单）
- `--delay-request`：请求间隔毫秒数
- `--timeout-request`：请求超时毫秒数

### CI/CD 集成示例

在 `.github/workflows/test.yml` 中添加：

```yaml
- name: Postman API Tests
  run: |
    npm install -g newman
    newman run postman/ticket-system.postman_collection.json \
      --reporters cli \
      --suppress-exit-code 1
```text

## 9.5.7 测试 WebSocket 连接

Postman 也支持 WebSocket 测试：

1. 在 Postman 中选择 **File → New → WebSocket Request**
2. 输入 URL: `ws://localhost:8000/ws/seats/1`
3. 添加 Header: `Origin: http://localhost:5173`
4. 点击 **Connect**
5. 发送消息：

```json
{"type": "subscribe", "event_id": 1}
```

**预期响应**：收到座位状态更新消息

```json
{"type": "seat_update", "data": {"available": 998, "total": 1000}}
```text

## 9.5.8 常见问题与排错

| 问题 | 原因 | 解决方法 |
| ------ | ------ | --------- |
| 请求返回 401 | Token 未设置或已过期 | 重新运行登录请求，检查 `{{token}}` 变量值 |
| 创建活动返回 403 | 当前用户不是 admin | 修改用户 role 为 admin，或使用 admin 账号 |
| 创建订单返回 400 | 活动未开始/已结束/库存不足 | 检查 `start_date` 和 `end_date` 设置 |
| 支付返回 400 | 订单号错误或订单已支付 | 核对 `order_no` 变量值 |
| 请求返回 422 | 请求体格式错误 | 检查 JSON 字段名和类型是否匹配 Pydantic schema |
| Newman 运行失败 | 环境变量未设置 | 确保 Postman Environment 文件中有 `base_url` 初始值 |
